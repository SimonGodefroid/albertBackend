Les routes à utiliser sont:

Pour le front

Events

renvoie les événements triés par date décroissante (d'aujourd'hui jusqu'au plus éloigné dans le temps), pour une catégorie donnée.
http://localhost:3002/api/events/?category=5 Test

Pour le dev (à utiliser pour les tests par le back uniquement)

Dates Filter Dev

Route pour checker que les events sont bien dans l'ordre, les in progress en premier puis tous les events.
http://localhost:3002/api/events/date/filtersdev Test
